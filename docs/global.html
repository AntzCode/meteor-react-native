open()

Makes `open` a no-op if there's already a `rawSocket`. This avoids memory / socket leaks if `open` is called twice (e.g. by a user calling `ddp.connect` twice) without properly disposing of the socket connection. `rawSocket` gets automatically set to `null` only when it goes into a closed or error state. This way `rawSocket` is disposed of correctly: the socket connection is closed, and the object can be garbage collected.
